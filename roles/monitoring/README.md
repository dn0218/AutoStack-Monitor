# Role: monitoring

## 中文说明
### 功能概述
部署监控系统，实现对容器运行状态的可视化。
* **Zabbix Agent 2**: 部署为容器，并映射宿主机 Podman Socket 以采集指标 。
* **Grafana**: 仅在 `node1` 部署，并预装 Zabbix 插件 。
* **演示优化**: 开启匿名登录模式（Admin 权限），方便演示环境即时查看仪表盘 。

### 为什么这样写？
* **Socket Mapping**: 将 `/run/user/.../podman.sock` 映射进容器是 rootless 监控的关键 。
* **Flush Handlers**: 在启动服务前强制执行 `flush_handlers`，确保 systemd 单元文件已由 Quadlet 成功生成 。

## English Description
### Overview
Deploys the monitoring system to provide visualization of container operational status.
* **Zabbix Agent 2**: Deployed as a container with the host's Podman Socket mapped to collect metrics.
* **Grafana**: Deployed only on `node1` with the Zabbix plugin pre-installed.
* **Demo Optimization**: Enables anonymous login (Admin privileges) to facilitate immediate dashboard access in demo environments.

### Rationale
* **Socket Mapping**: Mapping `/run/user/.../podman.sock` into the container is key for rootless monitoring.
* **Flush Handlers**: Forcing `flush_handlers` before starting services ensures that systemd unit files have been successfully generated by Quadlet.

### 故障排查与修复经验 (Troubleshooting & Fixes)
1. Systemd 用户实例通信故障 (Service Communication)
问题描述: Ansible 报错 Could not find the requested service: host。

根本原因: Ansible 通过 become 切换用户后丢失了环境变量，导致 systemd 模块默认回退到系统全局范围查找，而非用户空间。

修复方法: 在任务中显式注入 XDG_RUNTIME_DIR 和 DBUS_SESSION_BUS_ADDRESS，明确指定目标 D-Bus 通信路径。

2. Quadlet 服务同步延迟 (Service Synchronization)
问题描述: 配置已分发但服务未被 systemctl 识别。

根本原因: Quadlet 的 .container 文件需要经由 podman-systemd-generator 转换，daemon-reload 是异步或需要时间触发的。

修复方法: 使用 ansible.builtin.meta: flush_handlers 强制在启动服务前完成重载，并配合适当的 pause 等待文件生成。

3. 容器运行时权限与冲突 (Runtime Conflicts)
端口抢占: 宿主机若运行原生 Zabbix Agent，会占用 10050 端口。修复: 统一将容器映射端口设为 10051:10050。

用户命名空间异常: 在 Rootless 环境下手动指定 User/Group 会导致 UID 映射冲突引发崩溃（exit-code）。修复: 移除模板中的 User/Group 定义，由 Podman 默认处理映射。

环境变量缺失: 官方 zabbix-agent2 镜像在未配置 ZBX_SERVER_HOST 时会立即退出。修复: 在 Quadlet 配置中添加 Environment=ZBX_SERVER_HOST=...。

### Troubleshooting & Fixes (English)
1. Systemd User Instance Communication
Issue: Ansible reports Could not find the requested service: host.

Root Cause: Ansible loses environment variables after become, causing the systemd module to default to the system scope instead of the user space.

Fix: Explicitly inject XDG_RUNTIME_DIR and DBUS_SESSION_BUS_ADDRESS into tasks to target the correct D-Bus communication path.

2. Quadlet Lifecycle Management
Issue: Files are deployed but not recognized by systemctl.

Root Cause: .container files require a daemon-reload to trigger the podman-systemd-generator.

Fix: Use ansible.builtin.meta: flush_handlers to force a reload, followed by a brief pause to ensure unit files are generated.

3. Runtime Permissions & Conflicts
Port Conflicts: Host native services may claim port 10050. Fix: Use 10051:10050 as the mapping.

Namespace Mismatch: Manually specifying User/Group in rootless mode causes UID mapping failures. Fix: Remove hardcoded UID/GID settings to allow default mapping.

Environment Configuration: Official images exit immediately if ZBX_SERVER_HOST is undefined. Fix: Add Environment variables in the Quadlet config.
